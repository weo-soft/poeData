For general information on contributing datasets, please refer to the Contribution page.

There is one approach to collect data on weightings:

  • Using a Vendor recipe (5-to-1)

5-to-1 Vendor recipe

When using the Vendor recipe, Contracts must be rare before vendoring. The vendored Contracts must be of the same Contract type (eg. Bunker, Underbelly or Repository). When using different Contract types as inputs, separate the results and treat them as different Datasets (one Dataset per Contract type). You will receive a Contract with a random job type among the jobs that are viable for that Contract type.

  1. Choose Your Input Contract Type: Select a specific Contract type to use as input
  2. Make Contracts Rare: Ensure all Contracts you vendor are rare (use Orbs of Alchemy or other means)*
  3. Vendor Recipe: Sell 5 rare Contracts of that type to a vendor to receive 1 Contract back with a random job type (from the jobs viable for that Contract type)
  4. Record Results: Note exactly which Contract type you used as input and the jobs the received Contracts have.
  5. Continue Collecting: The more batches you complete, the more accurate your dataset will be

Note: It is recommended to collect at least 1000+ vendore contracts for basic results, and 10000+ for better accuracy.

Note: *Items with the same name in the same batch, are treated as "mirrored" and will not result in a new Contract. I you happen to have two Contracts with the same name, swap them in between Batches..
