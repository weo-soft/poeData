How to Contribute Datasets

Why Your Contributions Matter

To accurately calculate item weightings and probabilities, we need as many datasets as possible. Each dataset you contribute helps improve the accuracy of weight calculations for the entire community.

More datasets = More accurate calculations = Better data for everyone!

Dataset Structure

Your dataset should include the following information:

  • Name: A descriptive name for your dataset
  • Description: Optional description of the dataset
  • Date: The date when the data was collected (format: YYYY-MM-DD)
  • Patch: The game patch version (e.g., "3.26.0")
  • Sources: Where the data came from (Discord threads, forum posts, etc.)
  • Input Items: The items you used as input
  • Items: The output items with their counts

Quality Guidelines

To ensure your dataset is useful for weight calculations, follow these quality standards:

Minimum Sample Size

  • Recommended: At least 100+ input items for meaningful statistical results
  • Minimum: 50+ input items for basic calculations
  • Ideal: 500+ input items for highly accurate weight estimates
  • Why it matters: Larger sample sizes reduce statistical uncertainty and improve calculation accuracy

Data Accuracy

  • Count Accuracy: Ensure all item counts are accurate and verified
  • No Missing Data: Record all output items, including those with zero counts
  • Consistent Methodology: Use the same data collection approach throughout
  • Verification: Double-check your counts before submission

Completeness Requirements

  • All Output Items: Include every possible output item, even if count is 0
  • Complete Metadata: Provide name, description, date, patch version, and sources
  • Source Documentation: Clearly document where data came from (Discord, forum, personal collection)
  • Input Items: Specify exactly which items were used as input

Data Quality Checklist

Before submitting, verify your dataset has:

  • At least 50+ input items (100+ recommended)
  • Accurate counts for all output items
  • All possible output items included (even with 0 count)
  • Complete metadata (name, date, patch, sources)
  • Clear source documentation
  • Consistent data collection methodology
